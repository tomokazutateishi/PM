# アプリ開発向けマルチエージェント構成（MULTI_AGENTS_APP_DEVELOPMENT.md）

**版:** v1.0  
**目的:**  
アプリ開発プロジェクトに特化したマルチエージェント構成。各エージェントが独立した役割を担い、協調してアプリ開発を支援する。

---

## 1) アプリ開発向けエージェント構成の概要

### 1.1 基本方針
- **開発フェーズ対応**: 要件定義→設計→開発→テスト→デプロイの各フェーズに対応
- **技術領域特化**: 各エージェントが専門技術領域を担当
- **独立性**: 各エージェントは独自の判断でタスクを実行可能
- **並列実行**: 設計・開発・テストを並列で進められる構成

### 1.2 エージェント一覧（アプリ開発特化版）

#### プロジェクト管理系（既存）
| エージェントID | 役割名 | 主な責任範囲 | 独立実行可能 |
|---|---|---|---|

## 2) アプリ開発特化エージェントの詳細定義

### 2.1 要件定義エージェント（agent-requirements）

**役割定義:**
アプリの機能要件・非機能要件を定義し、ユーザーストーリーを作成する。

**独立実行可能タスク:**
- 機能要件の定義（FR-001, FR-002...）
- 非機能要件の定義（NFR-001, NFR-002...）
- ユーザーストーリーの作成
- 要件定義書の作成・更新
- 要件トレーサビリティマトリクスの作成
- 受入基準の定義

**出力形式:**
- 要件定義書（Markdown）
- ユーザーストーリー一覧
- 要件トレーサビリティマトリクス（CSV/Markdown）

**保存先:** `/{ProjectName}/02_要件定義/`

**他エージェントとの連携:**
- アーキテクチャ設計に要件を提供
- UI/UXデザインに機能要件を提供
- テストエージェントに受入基準を提供

---

### 2.2 アーキテクチャ設計エージェント（agent-architecture）

**役割定義:**
システムアーキテクチャの設計と技術選定を担当。

**独立実行可能タスク:**
- システムアーキテクチャの設計
- 技術スタックの選定
- データベース設計
- API設計
- 設計パターンの選定
- アーキテクチャ図の作成

**出力形式:**
- アーキテクチャ設計書（Markdown）
- 技術選定書
- データベース設計書（ER図含む）
- API設計書（OpenAPI仕様含む）
- アーキテクチャ図（Mermaid/PlantUML）

**保存先:** `/{ProjectName}/03_設計/アーキテクチャ/`

**他エージェントとの連携:**
- 要件定義から要件を受領
- 開発エージェントに設計を提供
- DevOpsエージェントにインフラ要件を提供

---

### 2.3 UI/UXデザインエージェント（agent-uiux）

**役割定義:**
ユーザーインターフェースとユーザー体験の設計を担当。

**独立実行可能タスク:**
- ワイヤーフレームの作成
- デザインシステムの定義
- プロトタイプの作成
- ユーザーフローの設計
- アクセシビリティの考慮
- デザインガイドラインの作成

**出力形式:**
- ワイヤーフレーム（Markdown/図表）
- デザインシステム仕様書
- プロトタイプ（Figmaリンク/画像）
- ユーザーフロー図
- デザインガイドライン

**保存先:** `/{ProjectName}/03_設計/UIUX/`

**他エージェントとの連携:**
- 要件定義から機能要件を受領
- 開発エージェントにデザイン仕様を提供
- テストエージェントにUIテスト基準を提供

---

### 2.4 開発エージェント（agent-development）

**役割定義:**
アプリのコーディングと実装を担当。

**独立実行可能タスク:**
- コードの実装
- コードレビュー
- リファクタリング
- パフォーマンス最適化
- バグ修正
- コード品質の監視

**出力形式:**
- ソースコード
- コードレビュー記録
- 実装メモ
- 技術的負債リスト

**保存先:** `/{ProjectName}/04_開発/`

**他エージェントとの連携:**
- アーキテクチャ設計から設計を受領
- UI/UXデザインからデザイン仕様を受領
- テストエージェントにテスト対象を提供
- セキュリティエージェントと連携してセキュリティ対策を実装

---

### 2.5 テストエージェント（agent-testing）

**役割定義:**
テスト計画・テストケース作成・テスト実行を担当。

**独立実行可能タスク:**
- テスト計画の作成
- テストケースの作成
- 単体テストの作成・実行
- 結合テストの計画・実行
- E2Eテストの計画・実行
- テストレポートの生成
- バグレポートの作成

**出力形式:**
- テスト計画書（Markdown）
- テストケース一覧（CSV/Markdown）
- テストレポート
- バグレポート

**保存先:** `/{ProjectName}/05_テスト/`

**他エージェントとの連携:**
- 要件定義から受入基準を受領
- 開発エージェントからテスト対象を受領
- 品質管理エージェントにテスト結果を報告

---

### 2.6 DevOps/インフラエージェント（agent-devops）

**役割定義:**
CI/CDパイプライン・インフラ構築・デプロイを担当。

**独立実行可能タスク:**
- CI/CDパイプラインの構築
- インフラ構築（IaC）
- デプロイ自動化
- 環境管理（開発/ステージング/本番）
- モニタリング設定
- ログ管理設定
- バックアップ戦略の策定

**出力形式:**
- CI/CD設定ファイル（YAML/JSON）
- インフラ構成図
- デプロイ手順書
- 環境設定ドキュメント

**保存先:** `/{ProjectName}/06_インフラ/`

**他エージェントとの連携:**
- アーキテクチャ設計からインフラ要件を受領
- 開発エージェントと連携してCI/CDを構築
- セキュリティエージェントと連携してセキュリティ設定を実装

---

### 2.7 セキュリティエージェント（agent-security）

**役割定義:**
セキュリティ対策と脆弱性分析を担当。

**独立実行可能タスク:**
- セキュリティ要件の定義
- 脆弱性分析
- セキュリティテストの実施
- セキュリティ対策の実装支援
- セキュリティインシデント対応計画
- セキュリティ監査

**出力形式:**
- セキュリティ要件書
- 脆弱性レポート
- セキュリティテストレポート
- セキュリティ対策チェックリスト

**保存先:** `/{ProjectName}/07_セキュリティ/`

**他エージェントとの連携:**
- 要件定義にセキュリティ要件を追加
- 開発エージェントにセキュリティ対策を指導
- DevOpsエージェントにセキュリティ設定を提供

---

### 2.8 ドキュメントエージェント（agent-documentation）

**役割定義:**
技術文書・API仕様書・マニュアルの作成を担当。

**独立実行可能タスク:**
- API仕様書の作成（OpenAPI/Swagger）
- 技術文書の作成
- ユーザーマニュアルの作成
- 開発者向けドキュメントの作成
- ドキュメントの更新・保守

**出力形式:**
- API仕様書（OpenAPI/Swagger）
- 技術文書（Markdown）
- ユーザーマニュアル（Markdown/PDF）
- 開発者ガイド

**保存先:** `/{ProjectName}/08_ドキュメント/`

**他エージェントとの連携:**
- アーキテクチャ設計から設計情報を受領
- 開発エージェントから実装情報を受領
- 要件定義から機能情報を受領

## 3) 開発フェーズ別のエージェント連携

### 3.1 要件定義フェーズ

```
agent-requirements (主導)
  ↓ 要件を提供
agent-architecture, agent-uiux, agent-security
```

### 3.2 設計フェーズ

```
agent-architecture (システム設計)
agent-uiux (UI/UX設計)
  ↓ 設計を提供
agent-development, agent-testing, agent-devops
```

### 3.3 開発フェーズ

```
agent-development (実装)
  ↓ 並列実行
agent-testing (テスト作成)
agent-documentation (ドキュメント作成)
agent-security (セキュリティ対策)
```

### 3.4 テストフェーズ

```
agent-testing (テスト実行)
  ↓ 結果を報告
agent-quality (品質評価)
agent-development (バグ修正依頼)
```

### 3.5 デプロイフェーズ

```
agent-devops (デプロイ)
agent-security (セキュリティ確認)
agent-testing (本番環境テスト)
```

## 4) 実行例

### 例1: 要件定義の独立実行

```
ユーザー: 
@agent-requirements アプリの要件定義書を作成してください。
機能要件と非機能要件を定義し、ユーザーストーリーを作成してください。

agent-requirements:
1. プロジェクト情報を確認
2. 機能要件を定義（FR-001, FR-002...）
3. 非機能要件を定義（NFR-001, NFR-002...）
4. ユーザーストーリーを作成
5. 要件定義書を生成
6. 他エージェントに要件を共有
```

### 例2: 設計フェーズの並列実行

```
ユーザー: 
以下のエージェントに並列でタスクを実行してください：

1. @agent-architecture: システムアーキテクチャを設計
2. @agent-uiux: UI/UXデザインを作成
3. @agent-security: セキュリティ要件を定義

各エージェントは要件定義書を参照して、独立して作業してください。
```

### 例3: 開発・テストの協調実行

```
ユーザー: 
@agent-development 機能を実装してください。
@agent-testing 実装された機能のテストケースを作成し、テストを実行してください。

両エージェントは協調して作業し、テスト結果を共有してください。
```

## 5) 技術スタック別の考慮事項

### 5.1 Webアプリ開発
- **フロントエンド**: React, Vue, Next.js等
- **バックエンド**: Node.js, Python, Go等
- **データベース**: PostgreSQL, MySQL, MongoDB等
- **インフラ**: AWS, GCP, Azure等

### 5.2 モバイルアプリ開発
- **iOS**: Swift, SwiftUI
- **Android**: Kotlin, Jetpack Compose
- **クロスプラットフォーム**: React Native, Flutter

### 5.3 デスクトップアプリ開発
- **Electron**: Web技術を使用
- **ネイティブ**: C++, C#, Java等

---

## 6) 注意事項

- **独立性の維持**: 各エージェントは自身の責任範囲に集中
- **技術的負債の管理**: 開発エージェントが技術的負債を記録・報告
- **セキュリティの早期考慮**: セキュリティエージェントを早期から参画
- **ドキュメントの継続的更新**: ドキュメントエージェントが常に最新状態を維持

---

## 7) 次のステップ

1. 各エージェント用の個別設定ファイルを作成
2. プロジェクトの技術スタックに合わせてエージェントをカスタマイズ
3. 開発フェーズに応じたエージェント連携ルールを定義
4. 実際のプロジェクトで試行・改善
