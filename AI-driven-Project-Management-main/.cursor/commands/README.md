# Cursor Custom Commands トラブルシューティングガイド

## 問題
`/` を入力しても `aipjm/XX_phase/YY_document_name` の形式でコマンドが表示されない

## 確認手順

### 1. Cursor の再起動
- Cursor を完全に終了して再起動してください
- コマンドファイルを追加/修正した後は、再起動が必要な場合があります

### 2. 設定の確認
- `Settings` → `Features` → `Chat` → `Custom commands` で有効になっているか確認
- または `Cmd/Ctrl + ,` で設定を開き、`Custom Commands` を検索

### 3. ファイル構造の確認
コマンドファイルは以下の構造になっている必要があります：
```
.cursor/
└── commands/
    └── aipjm/
        └── 01_initiating/
            └── 01_プロジェクト憲章.md
```

各ファイルの先頭には以下が必要です：
```markdown
# name: コマンド名
# description: コマンドの説明
```

### 4. ワークスペースの確認
- プロジェクトのルートディレクトリに `.cursor/commands/` が存在するか確認
- ワークスペースとして開いているディレクトリが正しいか確認

### 5. コマンドの実行方法
`/` を入力すると、以下のように表示されるはずです：
- `aipjm/01_initiating/01_プロジェクト憲章`
- `aipjm/01_initiating/02_ステークホルダー登録簿`
- `aipjm/02_planning/01_プロジェクトマネジメント計画書`
- など...

### 6. 代替方法
もし上記を試しても認識されない場合：
- コマンドを直接入力：`/aipjm/01_initiating/01_プロジェクト憲章`
- または、ファイルを直接開いてプロンプトをコピー＆ペースト

## 追加の確認事項
- ファイル名に特殊文字が含まれていないか
- ファイルの読み取り権限が適切に設定されているか
- Cursor のバージョンが最新かどうか
