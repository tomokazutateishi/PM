# クライアント会社概要＆情報

## 顧客対応体制

OBCは「奉行シリーズ」を提供する中堅・中小企業向け基幹業務パッケージベンダーであり、サポートセンターを事業の“要”と位置付けています。

- 東京（約200席）と札幌（約40席）の2拠点で運営
- 社内部門（開発・営業・マーケ等）もローテーションで対応
- 外部委託スタッフ約110名を含め、1日1,500～2,000件の問い合わせに対応
- 奉行 Netサービスへの会員登録（サービス購入者のみ）すれば、オンライン、電話、FAXでの詳細なサポートを受けられる
- 問い合わせ内容は操作方法から税制・IT相談まで多岐にわたる
- 顧客の画面をリモート共有しながら案内する支援体制
- 電話・FAX・Webフォームによる受付。今後はオンライン主軸へ移行予定
- 顧客の声を製品改善に活かす「製販保一体」の体制

### サポート拠点と対応件数

| 項目 | 内容 |
| --- | --- |
| サポート拠点数 | 東京・札幌の2拠点 |
| 対応席数 | 約240席（東京200、札幌40） |
| 1日あたり対応件数 | 1,500～2,000件 |
| 奉行シリーズ導入社数 | 56万社以上 |

## DXの取り組み

OBCは有人対応の強みを維持しつつ、デジタル技術による効率化を推進しています。

### 主なDX施策

- **電話対応DX**：NTTナビダイヤル導入により話し中件数などの通話データを可視化
- **カスタマーサクセス部門**：Gainsight導入によりクラウド利用状況を分析し、解約リスクを早期検知
- **奉行AIチャット**：生成AIによる対話型チャットボットを製品内に搭載（2024年10月リリース）
- **サポートポータル整備**：「奉行Netサービス」でFAQや制度改正情報を提供
- **リモートサポート**：顧客PC画面への遠隔接続による支援
- **マルチチャネル戦略**：電話・FAX・Webフォームに加え、将来的にチャット導入も視野

### DX施策の年表

| 年 | 施策 |
| --- | --- |
| 2015年 | ナビダイヤル導入による通話データの可視化 |
| 2021年 | CS推進室設立 |
| 2023年 | Gainsight導入によるリスク検知と自動対応 |
| 2024年 | 奉行AIチャットリリース |
| 2025年 | オンラインサポート主軸へ移行中 |

### 競合比較

主要競合とのサポート体制・DX活用の比較は以下の通りです。

### サポート体制比較表

| 企業 | サポート手段・特徴 |
| --- | --- |
| OBC（奉行） | 電話・Webフォーム・FAX・リモート。専任スタッフによる伴走支援。製品内AIチャット搭載。 |
| 弥生 | 電話中心。チャットボット導入済み。有人チャット対応あり。 |
| PCA | 電話・メール中心。訪問サポート可能。チャット導入なし。 |
| freee | チャット・メール中心。AIチャット24h＋有人チャット。電話は上位プラン限定。 |
| マネーフォワード | チャット・メール・電話対応。AI＋有人チャットのハイブリッド型。 |

### DX活用の特徴

- **OBC**：有人対応を維持しつつ、AIチャット・Gainsightなどで効率化
- **freee/MF**：クラウドネイティブでAI活用が進んでおり、自動化重視
- **弥生/PCA**：対人対応を重視しつつ、徐々にチャットボット等を導入
